# AI継続プロンプト - NDI-Telopプロジェクト

## プロジェクト概要

このプロジェクトは、Avalonia UIフレームワークとC#/.NET 8.0を使用して、NDI (Network Device Interface) 経由でテロップ（テキストオーバーレイ）グラフィックを作成・ブロードキャストするデスクトップアプリケーション「NDI-Telop」を開発するものです。

## 現在の進捗状況

### 完了した機能

- SkiaSharpレンダリングによる基本的なNDI出力機能
- リアルタイムプレビューシステム
- プリセット管理システム（プリセットの作成、編集、切り替え）
- トランジション効果（フェードイン/アウト、スライドアニメーション）
- 高度なエディタ機能：フォントファミリー選択、カラーピッカー
- 画像オーバーレイサポート（ロゴ、グラフィック）
- 自動クリア機能（指定秒数後のテロップ自動消去）
- NDIランタイムのインストールチェックとエラーハンドリング（スタブ化済み）

### 最近の作業

- `NdiService.cs` におけるNDIlib型参照の問題を修正
- `MainWindowViewModel` の警告を修正
- アプリケーションのビルドとテストを成功裏に実施
- すべての変更をGitHubの `main` ブランチにマージ済み
- **NDIランタイムのスタブ化**: `NdiTelop/src/NdiTelop/Services/NdiService.cs` を修正し、NDIランタイムがインストールされていない環境でもビルドが成功するようにしました。NDI機能は無効化されますが、開発の継続が可能です。
- **環境構築スクリプトの作成**: `setup.sh` を作成し、新しい開発環境でのセットアップを自動化しました。このスクリプトは、.NET SDKのインストール、Gitのインストール、リポジトリのクローン、プロジェクトのビルドを行います。
- **GitHub Actionsのセットアップ**: `.github/workflows/dotnet.yml` を作成し、プッシュおよびプルリクエスト時に自動的にビルドとテストが実行されるようにCI/CDパイプラインを設定しました。

## 技術スタック

- **フレームワーク**: Avalonia UI (クロスプラットフォーム .NET デスクトップフレームワーク)
- **言語**: C# (.NET SDK 8.0)
- **グラフィックス**: SkiaSharp
- **NDI**: NewTek.NDI ライブラリ (NDIランタイムのスタブ化により、ランタイムなしでもビルド可能)
- **バージョン管理**: GitHub (rnakamura1997/NDI-Telop)

## アーキテクチャ

- MVVMパターン (Model-View-ViewModel)
- サービス層 (`NdiService`, `RenderService`)
- プリセットベースの構成システム

## 主要ファイル

- `NdiTelop.csproj`: メインプロジェクトファイル。NuGetパッケージ参照を含む。
- `src/NdiTelop/Services/NdiService.cs`: NDI出力機能とNDIランタイム統合を処理。NDIランタイムのスタブ化が適用済み。
- `src/NdiTelop/Services/RenderService.cs`: SkiaSharpを使用してテロップグラフィックのレンダリングを管理。
- `src/NdiTelop/ViewModels/MainWindowViewModel.cs`: UIロジックとプリセット管理を処理するメインビューモデル。
- `setup.sh`: 新しい環境でのプロジェクトセットアップを自動化するスクリプト。
- `.github/workflows/dotnet.yml`: GitHub ActionsのCI/CDワークフロー定義ファイル。

## 残りの開発タスク (優先順位順)

1.  **オーバーレイ管理UI** (高優先度): UIからオーバーレイアイテムを動的に追加/編集/削除する機能。
2.  **設定の永続化** (中優先度): プリセットとNDI設定をJSONファイルに保存/ロードする機能。
3.  **アルファチャンネルの最適化** (中優先度): NDIの透明度が正しく機能することを確認し、必要に応じて最適化。
4.  **エラーハンドリングの強化** (中優先度): NDIランタイムが見つからない場合などに、ユーザーフレンドリーなダイアログを表示。
5.  **ドキュメントの更新**: `README.md` を含む、すべての新しい機能に関するドキュメントを更新。
6.  **コードクリーンアップ**: 未使用コードの削除、警告の修正。
7.  **リリースビルド**: プラットフォーム固有のビルド（Windows/Linux/macOS）を設定。

## 次のAIエージェントへの指示

このプロジェクトは、NDIランタイムのスタブ化、環境構築スクリプト (`setup.sh`)、およびGitHub ActionsのCI/CD設定が完了しており、他のAIエージェントが開発を継続しやすい状態になっています。

**以下の手順で作業を開始してください:**

1.  **リポジトリのクローン**: `git clone https://github.com/rnakamura1997/NDI-Telop.git`
2.  **環境構築**: クローンしたディレクトリ内で `./setup.sh` を実行し、開発環境をセットアップします。
3.  **コードの確認**: `src/NdiTelop/Services/NdiService.cs` の変更点を確認し、NDIランタイムがない場合の挙動を理解してください。
4.  **GitHub Actionsの確認**: `.github/workflows/dotnet.yml` を確認し、CI/CDの仕組みを理解してください。
5.  **開発の継続**: 上記「残りの開発タスク」リストの**「オーバーレイ管理UI」**から開発を再開してください。UIの設計と実装に焦点を当て、ユーザーが直感的にオーバーレイを操作できるようなインターフェースを目指してください。
6.  **テストの実施**: 変更を加えるたびに、`dotnet build` および `dotnet test` を実行し、必要に応じて新しいテストを追加してください。GitHub Actionsが自動的にテストを実行しますが、ローカルでの確認も重要です。
7.  **コミットとプッシュ**: 作業は定期的にコミットし、適切なコミットメッセージを付けてGitHubにプッシュしてください。新しい機能や大きな変更は、新しいブランチを作成して作業し、プルリクエストを通じて `main` ブランチにマージすることを推奨します。

不明な点があれば、このドキュメントを参照するか、必要に応じてユーザーに質問してください。
