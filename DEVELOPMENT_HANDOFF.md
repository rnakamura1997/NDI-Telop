# NDI-Telop プロジェクト開発引き継ぎドキュメント

## 1. プロジェクト概要

NDI-Telop は、NewTek NDI (Network Device Interface) を使用してリアルタイムでテロップを生成・送出するためのデスクトップアプリケーションです。Avalonia UI フレームワークと .NET 8 を使用して開発されています。主な目的は、放送やライブ配信において、動的なテロップ表示を容易にすることです。

## 2. 現在のプロジェクト状況

Phase 1b までの主要な機能（リアルタイムプレビュー、NDI送出、トランジション、高度なエディタ機能、オーバーレイ画像描画、自動クリア機能）の実装が完了し、GitHub の `main` ブランチにマージされています。プロジェクトはビルド可能で、基本的な機能は動作します。

## 3. 実装済み機能

-   **NDI 初期化と送出**: 指定されたソース名で NDI ストリームを初期化し、プログラム出力とプレビュー出力の両方を送出できます。
-   **リアルタイムプレビュー**: UI 上で現在のテロップ内容をリアルタイムで確認できます。
-   **テロップ表示**: テキスト、フォント、色、背景色などを設定し、テロップとして表示できます。
-   **トランジション**: プリセット切り替え時にフェードイン/アウトなどのアニメーション効果を適用できます。
-   **高度なエディタ機能**: フォントファミリーの選択、カラーピッカーによる色選択が可能です。
-   **画像オーバーレイ**: テロップ上に画像をオーバーレイ表示できます。
-   **自動クリア**: 指定した秒数後にテロップを自動的にクリアする機能が実装されています。

## 5. プロジェクト構造の概要

-   `/src/NdiTelop`: メインアプリケーションのソースコード。
    -   `/Models`: データモデル（Preset, NdiConfig, OverlayItem など）。
    -   `/Views`: Avalonia UI の XAML ファイル（MainWindow.axaml, PreviewCanvas.axaml など）。
    -   `/ViewModels`: UI のロジックとデータバインディングを担う ViewModel（MainWindowViewModel など）。
    -   `/Services`: ビジネスロジックと外部サービス連携（NdiService, RenderService など）。
    -   `/Interfaces`: サービス層のインターフェース。
-   `/tests/NdiTelop.Tests`: 単体テストプロジェクト。
-   `/web`: 将来的なWebインターフェースのプレースホルダー。

**主要なファイルと役割:**

-   `src/NdiTelop/Services/NdiService.cs`: NDI の初期化、フレーム送出、アクティブ状態管理。
-   `src/NdiTelop/Services/RenderService.cs`: テロップの描画ロジック、トランジション、オーバーレイのレンダリング。
-   `src/NdiTelop/ViewModels/MainWindowViewModel.cs`: メインウィンドウのデータコンテキスト。UI操作とビジネスロジックの連携。
-   `src/NdiTelop/Views/MainWindow.axaml`: メインウィンドウのUI定義。
-   `src/NdiTelop/Controls/PreviewCanvas.axaml.cs`: プレビュー表示用のカスタムコントロール。

## 6. ビルドと実行手順

1.  **NDI SDK のインストール**: NewTek NDI SDK を開発環境にインストールしてください。
2.  **リポジトリのクローン**: `git clone https://github.com/rnakamura1997/NDI-Telop.git`
3.  **依存関係の復元**: プロジェクトルートで `dotnet restore` を実行。
4.  **ビルド**: プロジェクトルートで `dotnet build` を実行。
5.  **実行**: `dotnet run --project src/NdiTelop`

## 7. 既知の課題と注意事項

-   NDIlib の型参照に関するコンパイルエラーが頻発したため、`using static NewTek.NDIlib.NDIlib;` を使用して参照を単純化しています。もし問題が発生した場合は、完全修飾名での参照に戻すことを検討してください。
-   `NdiService.cs` 内の `NDIlib.IsSupported` チェックは、NDIランタイムがインストールされているかを確認するためのものです。これが `false` の場合、NDI機能は利用できません。
-   `Avalonia.Controls.ColorPicker` パッケージは、一時的に追加されましたが、最終的なUI統合のために調整が必要です。
